# Specification: Group Project Coordinator

## Goal
Facilitate student group projects by clearly delegating tasks, establishing timelines, ensuring fair workload distribution, and providing communication structure to minimize dysfunction and resentment.

## Requirements
1. **Task Breakdown**: Decompose project into clear, manageable tasks
2. **Fair Distribution**: Balance workload by effort, time, and skill required
3. **Skill Matching**: Assign tasks to members' strengths when possible
4. **Accountability Structure**: Clear ownership and deadlines per person
5. **Dependency Mapping**: Identify tasks that block other work
6. **Timeline**: Realistic schedule with milestones and buffers
7. **Communication Plan**: How often to meet/check-in, what platform to use
8. **Conflict Resolution**: Ground rules and process for disagreements
9. **Progress Tracking**: Simple system to show status and blockers
10. **Contingency Planning**: Backup plans if member doesn't deliver

## Input Format
- Project description and requirements
- Team members and their skills/availability
- Project deadline
- Required deliverables
- Any constraints (format, length, requirements)
- Team size

## Output Format
Project coordination plan with:
- Complete task list with effort estimates
- Task assignments by team member
- Project timeline with milestones
- Dependency chart (what needs to happen first)
- Meeting schedule and communication plan
- Individual workload summary (fairness check)
- Progress tracking template
- Ground rules and expectations
- Contingency plans for common issues
- Final assembly checklist

## Quality Criteria
- Workload is equitably distributed
- Tasks are clearly defined and sized appropriately
- Dependencies are managed
- Timeline is realistic with buffer
- Reduces typical group project dysfunction
